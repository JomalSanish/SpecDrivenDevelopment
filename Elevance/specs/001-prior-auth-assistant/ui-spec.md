# UI Specification: Prior Authorization Evidence Assistant

## 1. Persona-Based Navigation Model

The application uses a React + TypeScript frontend. Upon login (via mocked Auth provider), the UI dynamically renders based on the user's role (1 of 7 personas).

### Queues & Dashboards
- **Intake Associate**: Sees the "Intake Queue". Focuses on cases with `status = Intake Review` or newly `Created` cases.
- **Nurse Reviewer**: Sees the "Nurse Queue". Focuses on clinical summaries and Gap Checklists.
- **Medical Director**: Sees the "Escalation Queue". Focuses on complex, ambiguous, or contradictory evidence flags.
- **Provider Relations**: Sees the "Pending Documentation Queue". Focuses on reviewing and sending drafted missing-document requests.
- **Operations Manager**: Sees a global "Queue Insights" dashboard. Views volume, SLAs, and routing accuracy.
- **Auditor & QA/Test Engineer**: Sees the "Audit Search" view. Can query any `case_id` for its full audit trail.

## 2. Case Detail View (The Reviewer Dashboard)

When a Nurse or Medical Director opens a case, the layout is structured for high data density:

### Left Panel: Case Summary & Policy
- **Member & Request Info**: Demographics, ICD-10, CPT codes.
- **Agent Summary**: The human-readable brief generated by the Summary Agent.
- **Routing Status**: Why this case is in the current queue (e.g., "Escalated: Contradictory evidence detected in clinical notes. Confidence: 0.89").

### Center Panel: Gap Checklist & Evidence Table
- **Policy Checklist**: A vertical list of criteria from the medical policy.
  - Each item shows a status chip: `[Present]`, `[Absent]`, or `[Unclear/Ambiguous]`.
- **Evidence Drill-down**: Clicking a checklist item expands a table below it showing the `EvidenceItem` array.
  - Columns: Source Document, Extracted Text, Confidence Score (color-coded red/yellow/green based on the `<0.6, 0.6-0.8, >0.8` thresholds).
  - Clicking the "Citation Ref" opens the Right Panel.

### Right Panel: Document Viewer
- Renders the original source document (PDF).
- Automatically scrolls to and highlights the exact `matched_text` referenced by the citation.

## 3. Audit Trail Drill-Down
- Accessible via a prominent "View Audit Log" button in the case header.
- Opens a modal or dedicated page showing a chronological timeline of `AuditLogEntry` records.
- Each entry displays: Timestamp, Agent Name, Model Version, Prompt Hash, and a raw JSON view of inputs/outputs.

## 4. Accessibility & Component Library
- **WCAG 2.1 AA Compliance**: Strict adherence to color contrast (especially for confidence score indicators), keyboard navigation for the data tables, and screen-reader compatible aria-labels for the checklist states.
- **Component Library**: Will utilize a data-dense UI library (e.g., MUI X-DataGrid, AG Grid, or similar tailored for enterprise operations) to handle large evidence tables efficiently.
